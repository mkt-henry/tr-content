# Gold Price 시세 화면 — 설계

날짜: 2026-06-08
프로젝트: ARIA Demo Studio — Treazer 데모 (`src/demos/treazer/gold-store/`)
상태: 승인됨

## 목적

`gold-store` 데모의 **v1 "포인트가 아니라 진짜 금"** variant 고도화.

현재 v1은 스토어 화면에서 미니 티커가 틱하며 골드 카드 환산값이 출렁이는 정도다.
여기에 전용 **Gold Price 시세 화면**을 추가해, 사용자가 실시간 금 시세를 직접 확인하고
**"내가 모은 포인트(골드)의 가치가 모은 시점보다 올라가고 있다"**는 자산감을 체감하게 한다.

전달 메시지의 주인공은 시세 차트 자체가 아니라 **내 골드의 평가손익(매입 대비 수익률)**이다.
주식 앱처럼 "내 골드 평가액 + 모은 시점 대비 +Y% ▲"를 화면 최상단에 크게 둔다.

## 화면 구조 — 스토어 → Gold Price 2단 플로우

```
[스토어 화면(기존)]
  My Gold Points 카드 ──탭──▶ [Gold Price 화면(신규)]  ──← 헤더 back──▶ [스토어]
```

단일 데모 안의 view 전환(`view: 'store' | 'price'`)으로 처리. 라우팅 추가 없음.

### 신규 Gold Price 화면 레이아웃 (위→아래)

```
┌─────────────────────────────────┐
│  ←   Gold Price                 │  헤더
├─────────────────────────────────┤
│ ★ 내 골드 평가 (다크 카드, 주인공)│
│   8,077 G                        │
│   ≈ ₫1,535  (0.0040 g)          │  현재 평가액 (시세 틱마다 ↑)
│   ┌───────────────────────────┐ │
│   │ 모은 시점 대비  +12.4% ▲   │ │  평가손익 — 초록, 크게
│   │ +₫169                     │ │
│   └───────────────────────────┘ │
├─────────────────────────────────┤
│  Gold Price        [Line|Candle]│
│  ₫152,400/g    ▲ 1.2%           │  현재 시세 + 등락
│  ▁▃▅▆▇ (우상향 캔들/라인 차트)  │
│  MA  BOLL  EMA                  │  (정적 표시)
│  [Now][1M][5M][1Y][ALL]         │  기간 토글
├─────────────────────────────────┤
│  Open  High  /  Low  Close      │  4분할 통계
├─────────────────────────────────┤
│  ⓘ About Gold Pricing           │
│  Conversion Rate: 1 G = …       │
└─────────────────────────────────┘
```

### 스크린샷(레퍼런스) 대비 의도적 차이

- 레퍼런스 차트는 **하락(▼0.72%)**이지만, 메시지가 "가치 상승"이므로 차트를 **전체 우상향 추세**의 더미 시계열로 만든다.
- 레퍼런스엔 없던 **평가손익 카드**를 최상단 주인공으로 추가한다.
- 레퍼런스의 USD/oz·THB 혼용 표기는 따르지 않고, **언어별 통화 단일 기준**으로 통일한다.

## 데이터 모델

### 평가손익 (데모 단순화)

- 보유 골드 `gold`(초기 8,077 G)는 고정 그램으로 환산: `grams = gold × GOLD_GRAMS_PER_UNIT`.
- **평균 매입 시세 `avgCost`** 도입 — 현재 시세보다 약 11% 낮게 잡아 "+12%대" 수익을 연출.
  (예: 현재 시세 ≈ 152,400/g 이면 avgCost ≈ 135,600/g)
- 현재 평가액 = `grams × goldPrice`, 언어별 통화로 환산.
- 수익률 = `goldPrice / avgCost − 1`. 평가이익 = 현재 평가액 − 매입 평가액.
- 시세 틱이 오를수록 평가액·수익률·평가이익이 함께 상승. `tickPrice()`의 상승 편향 유지/강화.
- 합계·환산값은 모두 계산으로 도출 — 하드코딩 금지.

### 차트 데이터

- 기간별(`Now / 1M / 5M / 1Y / ALL`) **우상향 추세 + 노이즈**의 더미 OHLC 시계열을 `data.ts`에 정의.
  각 기간은 점 개수/변동폭만 다르게 한다.
- 결정적(deterministic) 시드 기반으로 생성 — 매 렌더마다 차트가 달라지지 않도록(자동재생 안정성).
  `Math.random()` 직접 호출 대신 시드 기반 의사난수 또는 정적 배열.
- `Line`: area 라인. `Candle`: OHLC 캔들. 토글로 전환.
- `MA / BOLL / EMA` 보조지표와 `Open/High/Low/Close` 4분할은 **시각 요소로만 표시**, 인터랙션 없음
  (데모 정직성: 동작하지 않는 토글을 동작하는 척하지 않음).

### 통화/단위 일관성

- 기존 미니 티커의 하드코딩 `₩`를 포함해 **언어별 통화(`S$/¥/₫/฿`)로 통일**.
  시세는 `통화/g`, 평가액·평가이익도 동일 통화. 환산은 기존 `CURRENCY` 테이블 활용.
- 신규 UI 문자열은 4개 언어(`en/ja/vi/th`) 전부 `STR`에 추가.

## 차트 렌더링

- **의존성 없는 인라인 SVG 자체 구현** (recharts 미사용 — treazer 데모 컨벤션 일치, 캔들/연출 제어 용이).
- 작은 헬퍼 컴포넌트로 캔들/라인을 각각 렌더. 폰 프레임 안 사이즈에 맞춤.
- 자동재생 중 시세 틱에 맞춰 차트가 자라나는/갱신되는 연출 가능하도록 구성.

## 파일 구조

신규 화면이 커지므로 `screens.tsx` 비대화를 피해 분리한다.

- `GoldPriceScreen.tsx` (신규): Gold Price 화면 + 차트/평가카드 헬퍼.
- `screens.tsx`: 기존 스토어 화면. `My Gold Points` 카드에 탭 → `openPrice()` 연결.
  view에 따라 스토어/시세 화면을 전환 렌더(`AppScreens`에서 분기).
- `state.ts`: `view: 'store' | 'price'`, `openPrice()/closePrice()`, `avgCost`, 평가 셀렉터 추가. `reset()`에 view·avgCost 포함.
- `data.ts`: `avgCost` 초기값, 기간별 차트 시계열, 신규 `STR` 문자열, About/Conversion 문구 추가.
- `Mobile.tsx` / `Desktop.tsx`: 기존 래퍼 유지(화면 분기는 `AppScreens` 내부에서).

## 시나리오 (v1 재구성)

`realGoldScenario`를 화면 진입 흐름으로 재구성:

1. 스토어에서 `gold-card` / `My Gold Points` 강조.
2. 카드 탭 → `openPrice()` → Gold Price 화면 진입.
3. 평가 카드(`my-valuation`) 강조.
4. 시세 틱 여러 번 — 평가액·수익률·평가이익이 함께 상승하는 걸 보여줌. 차트도 갱신.
5. "모은 시점 대비 +X% ▲" 영역 클로즈업 — 핵심 메시지 마무리.

`data-demo-id`: `my-valuation`, `valuation-return`, `gold-chart`, `price-now`, 기간 토글 등 시나리오 타깃 부여.

v2 `exchangeScenario`는 스토어 단독 흐름이므로 영향 없음(진입 시 `view: 'store'` 보장).

## 검증 포인트

- 4개 언어 전환 시 통화/문자열이 모두 올바르게 표기.
- 자동재생 중 평가액·수익률이 단조 상승(또는 우상향 편향)으로 체감.
- `reset()` 후 view가 `store`로, avgCost·시세가 초기값으로 복귀.
- `npm run build`(tsc --noEmit) 통과.
